Этот алгоритм использует ссылки на родителей для обхода дерева in-order без использования рекурсии. 
Он начинается с самого маленького узла, а затем переходит к следующему по величине, используя либо правого ребенка, либо родителя. 
Самое сложное здесь — правильно найти следующего по величине узел, используя ссылки на родителей. 
Временная сложность остаётся O(n), так как мы посещаем каждый узел ровно один раз. Пространственная сложность — O(1).
